Before & After: A SQL-Based Data Cleanup

Raw Data
Download Raw Data
Issues in Raw Dataset:
  • Duplicate records without a unique identifier.
  • Untrimmed or inconsistently cased string values (e.g., company, country).
  • Inconsistent category labels such as variants of "Crypto".
  • Country names containing trailing punctuation (e.g., United States.).
  • Dates stored as strings in non-standard formats.
  • Empty strings present in critical fields such as industry and total_laid_off.
Cleaned Data
Download Cleaned Data
Applied Cleaning Techniques:
  • Used CTE with ROW_NUMBER() to identify and remove duplicates.
  • Applied trimming and case standardization to string fields.
  • Standardized categorical values for consistency (e.g., unified under "Crypto").
  • Cleaned country field by removing invalid characters like periods.
  • Converted string-based dates to SQL DATE format using STR_TO_DATE().
  • Populated missing or blank entries using self-join based imputation logic.
